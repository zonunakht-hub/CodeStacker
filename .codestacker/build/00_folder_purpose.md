# 🏗️ BUILD FOLDER - What to Build (Spec Driven)

## Purpose
This folder contains the specifications for what needs to be built. These are the "what" docs — feature specifications, requirements, and success criteria.

## Contents
- `01_project_overview.md` — High-level project spec
- `02_phase_1_spec.md` — Phase-specific specs (add more phases as needed)
- `backlog/` — Future feature specs not yet scheduled

## Usage Rules
- All plans must reference these specs
- Completed specs move to backlog/ when done
- Never build features not in spec without approval
- Specs can be split into phases
